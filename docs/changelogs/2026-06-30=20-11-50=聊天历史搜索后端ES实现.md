# 聊天历史搜索（Elasticsearch）后端实现

## 变更摘要
实现聊天历史搜索后端：按关键词/群成员/时间搜索，支持单群与全局范围，结果可跳转。基于 Elasticsearch
检索，新增独立 Kafka 消费者同步索引，存量回填脚本，权限收敛到“仅加群后消息”。

## 动机 / 背景
落地《docs/plans/2026-06-30-message-search-elasticsearch-design.md》设计方案的后端部分。复用现有
Outbox→Kafka 链路，新增独立 consumer group 异步写 ES，与发消息主链路解耦，面向海量消息提升检索性能。

## 主要内容
- `internal/search/`：ES 查询构建（权限范围 + 状态/系统消息排除 + search_after + 高亮）、响应解析、
  ES REST 客户端（net/http，无新依赖）、事件索引处理 `HandleEvent`、Kafka 消费者 `RunConsumer`、
  存量 `_bulk` NDJSON 构建。均含单元测试。
- `internal/service/service_search.go`：`SearchMessages` 编排（范围查询 + 权限校验 + 执行 + 解析），
  注入式 hook 便于测试。
- `internal/repo/repo.go`：`ListUserGroupScopes`（用户可搜群 + 入群序号）、`ListMessagesAround`
  （跳转上下文窗口）、`ScanMessagesAfterID`（回填扫描）；`AddMember`/`ApproveJoinRequest` 记录
  `join_sequence`。
- `internal/api`：`GET /api/v1/search/messages` 接口；`messages` 接口新增 `aroundSequence` 分支；
  新增搜索错误码。
- `cmd/es-indexer`、`cmd/es-backfill` 两个新进程；`cmd/api` 装配搜索后端。
- `internal/config`：`ES_ENABLED`/`ES_ADDRS`/`ES_INDEX`/`ES_INDEXER_CONSUMER_GROUP`。
- `deploy/mysql/init.sql`：`group_member.join_sequence` 列。
- `deploy/elasticsearch/`：含 IK 插件的 ES Dockerfile + 索引 mapping。
- `docker-compose.yml`：新增 `elasticsearch`、`groupflow-es-indexer` 服务与 `es-data` 卷；API 增加 ES 环境变量。
- `backend/Dockerfile`：构建 es-indexer / es-backfill 二进制。

## 验证
- `go build ./...` 通过；`go test ./...` 全绿（search 包覆盖查询/解析/客户端/索引/回填逻辑）。
- ES/MySQL/Kafka 端到端需在带 Elasticsearch 的环境中验证。
