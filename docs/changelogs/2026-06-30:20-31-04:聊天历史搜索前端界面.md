# 聊天历史搜索前端界面与跳转

## 变更摘要
前端实现聊天历史搜索面板与结果跳转：支持按关键词/群成员/时间搜索，单群与全局两种入口，点击搜索
结果跳转到群聊对应消息并高亮定位。

## 动机 / 背景
落地《docs/plans/2026-06-30-message-search-elasticsearch-design.md》设计方案的前端部分，对接后端
`GET /api/v1/search/messages` 与 `aroundSequence` 上下文接口。

## 主要内容
- `components/MessageSearchPanel.tsx`：搜索面板（关键词 debounce、群成员下拉、时间范围、游标加载更多、
  竞态请求丢弃）。关键词高亮在客户端完成（React 转义，避免服务端 HTML 的 XSS 风险）。
- `pages/ChatPage.tsx`：左侧「全局搜索」入口、群聊头部「搜索」入口；`jumpTo` 加载目标消息上下文窗口
  并高亮，`backToLatest` 回到最新消息（跳转态）。
- `components/VirtualMessageList.tsx`：新增 `highlightSequence`，定位滚动并高亮目标消息。
- `api/groupApi.ts`、`types/index.ts`：`searchMessages`、`getMessagesAround` 接口与 `SearchHit/SearchResult` 类型。
- `styles.css`：搜索面板与跳转高亮样式。
- 新增 `frontend/.gitignore` 忽略 `node_modules/`、`dist/`。

## 验证
- `npm run build`（tsc 类型检查 + vite 生产构建）通过。前端无单元测试运行器，按现有约定以类型检查 +
  构建作为验证手段。
