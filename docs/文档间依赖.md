## 最核心的依赖链

真正开发时，最重要的是这条链：

```text
数据库设计
  ↓
HTTP API + WebSocket 协议
  ↓
消息发送与投递链路
  ↓
Redis Key + Kafka Topic
  ↓
大群投递
  ↓
前端状态管理
  ↓
压测方案
  ↓
开发任务拆解
```

一句话总结：

**产品原型决定做什么，技术架构决定怎么分层，数据库/WebSocket/HTTP 决定基础能力，消息链路/Redis/Kafka/大群投递决定高并发核心，前端文档决定用户交互，压测文档验证设计是否扛得住，开发计划把所有文档落成任务。**
